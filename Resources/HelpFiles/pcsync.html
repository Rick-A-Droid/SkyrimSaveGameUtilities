Directories

In the application's directory you will find the following directory structure:

XML_TRANSACTIONS
     XML_in
     XML_Out_Effects
     XML_Out_Ingredients

XML_in This is where the XML files exported from the Android App are placed.  Existing files will always be over-written.  So, be sure to rename or move any you want to keep.
XML_Out_Effects This is where you need to put the edited effects XML file you want to Import to the Android App.  You should only put one file at a time in this directory.  Otherwise, it sends the first file found.
XML_Out_Ingredients This is where you need to put the edited ingredients XML file you want to Import to the Android App.  You should only put one file at a time in this directory.  Otherwise, it sends the first file found.
How it works

When PC Sync XML is enabled, you can use the Android App to Export and Import XML files (see the Android App's help screens for more info):

Export -

When you use the Android App to export an Effects or Ingredients XML file, it is saved to the XML_in directory.  When you have made your changes, put the changed file in the appropriate XML_Out directory, and use the Android App's Import function.

If a file already exists in the XML_in with the same filename, it over-writes the existing file.  So be sure to move or rename any file(s) you want to keep.

Import -

When you tell the Android App to import an Effects or Ingredients file, it sends the PC a request that specifies that it wants an Effects or Ingredients file.  The request tells the PC program to get the file from the XML_Out_Effects or XML_Out_Ingredients directory. 

The PC program just sends the file from the chosen directory.  It does not parse the file.  It just sends it as a stream of bytes.  The filename is also sent in that stream.

Since the Android App does not know the name of the file you want to import, when it requests an Effects or Ingredients file, the PC application will send the first file it finds in the appropriate directory.  The Android App will save the imported file in the App's private storage using the same filename.

This is not the same directory the default XML files are in.  You cannot change or delete the default files.  You do not need to change the filename(s) unless you want to import more than one file and keep both.
